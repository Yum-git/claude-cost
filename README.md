# Claude コスト可視化（claude-cost）

Claude Code のサブスク使用状況を、特別なコマンドを打たずに **視覚的に・常時** 確認するための VSCode 拡張機能です。以下を可視化します。

1. **サブスク使用制限の使用率（%）** — 公式 CLI（`claude -p "/usage"`）が返す **サーバー側の実測値**（5時間ウィンドウ・週次）とリセット時刻
2. **API 換算コスト** — サブスクで使った分を、もし API で同じだけ使っていた場合の金額（USD）。ローカルのセッションログから算出
3. **追加使用（超過）コスト** — 5時間ウィンドウが上限（100%）に到達したことを観測した時刻以降に使ったトークンの API 換算コスト、および上限到達の通知

> 本拡張自身は **OAuth トークンや API を直接使用しません**。使用率は公式 `claude` CLI に取得させ、その標準出力のみを読み取ります（`~/.claude/.credentials.json` や Keychain にも触れません）。コスト算出はこのマシンのローカルセッションログのみを参照します。

> **開発状況**: 主要機能（データ取得・コスト計算・使用率(公式CLI実測)・超過検知・UI（ステータスバー＋詳細パネル＋通知）・自動更新（活動駆動＋条件付きポーリング）・差分キャッシュ）まで実装済みです。Claude Code がトランスクリプトへ追記したこと（＝実際に使われたこと）を検知して即時更新し、さらにウィンドウがフォーカス中または直近に活動があるときは取得間隔（既定5分）ごとにも自動更新します。使用率%の再取得は5分TTLでスロットルされます（取得失敗が続く場合は指数バックオフで再取得を抑制）。

> **免責事項**: 本ソフトウェアは MIT ライセンスのもと **無保証・自己責任** で提供されます。表示される金額は公開 API 価格に基づく **ローカル推定値** であり、実際の請求額とは異なります。本ソフトウェアの利用または利用不能から生じたいかなる損害についても、作者は一切の責任を負いません。詳細は [ライセンス](#ライセンス) を参照してください。

## 重要な制約（必ずお読みください）

- 使用率（%）は公式 `claude -p "/usage"` の実測値です。使用率の取得は **5分間隔（既定）でキャッシュ** し、その結果は **複数の VSCode ウィンドウ間で共有** します（ウィンドウ数ぶん `/usage` を叩いて claude 側の取得制限に達するのを防ぐため）。一時的に取得できないときは直近の取得値を一定時間（既定30分）そのまま表示します。取得失敗が続く場合は**指数バックオフ（30秒→最大5分。上限は取得間隔に連動）**で再取得の間隔を自動的に広げ、claude への過剰なアクセスを防ぎます（成功すると通常の5分間隔へ即座に戻り、手動更新はバックオフを無視して即時取得します）。`claude` が見つからず直近値も無い場合のみ使用率を表示しません（その場合は API 換算コストのみ表示）。
- 使用率%の取得には **`claude` CLI が PATH 上または既知の場所に存在する必要があります**（`claudeCost.claudeCliPath` で明示指定可）。
- **セキュリティ上の注意**: `claude` の自動解決は PATH と既知の設置場所（`/opt/homebrew/bin` やホーム配下の `~/.claude/local` 等）を探索し、最初に見つかった実行ファイルを起動します。これらの場所が第三者に書き換え可能な環境では意図しない実行ファイルが起動され得るため、信頼できる環境で利用し、必要に応じて `claudeCost.claudeCliPath` で実体を明示してください。
- 追加使用（超過）コストは、5時間ウィンドウが100%に到達したことを**観測した時刻**以降のトークンから算出します。観測タイミングはポーリング間隔ぶん（最大数分）の誤差を含む**概算**です。使用率が一時的に取得できず直近値を再利用している間は、超過判定にもその値が使われるため、誤差が最大30分程度まで広がり得ます。
- 金額は **公開 API 価格に基づくローカル推定値** であり、実際の請求額とは異なります。サブスク利用分は実際には API 課金されません（「もし API だったら」の参考値です）。コスト集計対象は **このマシンで実行した分のみ** です。
- API 換算コストの集計区間は、**5時間・週次は現在時刻からのローリングウィンドウ**、**当日はローカルタイムの 0 時起点**、日次グラフはローカル日付基準です。

## 主な機能

- ステータスバーに使用率%・状態色を常時表示（クリックでメニュー: 強制更新 / パネル表示 / 設定）
- 詳細パネル（Webview）で 使用率%（5時間/週次・リセット時刻）・API 換算コスト（5時間/週次/当日/累計）・追加使用コスト・モデル別内訳・日次グラフを表示
- 使用率が「接近」「上限到達」へ悪化したときのポップアップ通知 + ステータスバー強調
- 追加使用（超過）課金が有効か否かの状態表示（`~/.claude.json` 由来・参考）
- 起動時・設定変更時・手動更新に加え、Claude Code の活動（トランスクリプト追記）を検知して即時更新。さらに、ウィンドウがフォーカス中または直近に活動があるときは取得間隔（既定5分）ごとに自動更新（条件付きポーリング。VSCode 不使用かつ活動なしのときはポーリングしません）。差分のみ読む（インクリメンタル）ため毎回の全件読み込みを避ける。使用率%の再取得は5分TTLでスロットル（取得成功時。取得失敗が続く場合は指数バックオフで再取得間隔を自動拡大し、claude への過剰アクセスを防止）

## データソースとプライバシー

本拡張が参照するのは以下のみです。

| 用途 | ソース | 読み取る内容 |
| --- | --- | --- |
| 使用率%・リセット時刻（要件①） | `claude -p "/usage"` の標準出力 | 使用率%（5時間/週次）とリセット時刻のテキストのみ。**拡張は OAuth トークンに触れません**（公式クライアントが自身の認証で取得） |
| トークン使用量・コスト（要件②③） | `~/.claude/projects/<project>/<session>.jsonl` | アシスタント応答の `usage`（トークン数・モデル名・時刻）のみ |
| 追加使用課金の状態（参考） | `~/.claude.json` | 超過課金関連の数フィールドのみ |

- **会話本文（プロンプト・応答テキスト）は読み取らず、保持もしません。** 集計に必要なトークン数・モデル名・時刻のみを抽出します。
- 認証情報ファイル（`~/.claude/.credentials.json`）や Keychain は **一切読み取りません**。
- 抽出した数値はローカルでの集計・表示にのみ使用し、外部送信は行いません。
- 差分キャッシュはプロセス内メモリのみに保持し、ディスクへ永続化しません（VSCode 終了で破棄、次回起動時に再走査します）。

## 必要環境

- VSCode `1.85.0` 以上（拡張は VSCode 内蔵の Node ランタイム上で動作します）
- 使用率%表示には Claude Code の `claude` CLI が必要（PATH 上、または `claudeCost.claudeCliPath` で指定）。未インストールでも API 換算コストは表示されます
- 開発（ビルド）環境: VSCode `1.125.0` / Node.js `v24.13.0` / npm `11.6.2`

## セットアップ・ビルド

```bash
npm install        # 依存関係のインストール
npm run compile    # 開発ビルド（dist/extension.js を生成）
npm run build      # 本番ビルド（minify）
npm run check-types  # 型チェックのみ（出力なし）
npm run watch      # ファイル変更を監視して自動ビルド
```

## 開発（デバッグ実行）

1. VSCode でこのフォルダを開く
2. `npm install` を実行
3. `F5`（Run Extension）で Extension Development Host を起動
4. 別ウィンドウが開いたら、アクティビティバーの「Claude Cost」アイコン、またはコマンドパレットの `Claude Cost: 使用状況を表示` で動作を確認

## インストール（vsix）

```bash
npm run package                         # claude-cost-x.x.x.vsix を生成
code --install-extension claude-cost-0.0.1.vsix
```

> `package.json` の `publisher` は暫定値（`local`）、`repository` フィールドは未設定です（git リポジトリ確定後に追加を推奨。現状はパッケージ時に `--allow-missing-repository` で省略しています）。ローカルインストールには支障ありませんが、Marketplace 等で配布・公開する場合は設定してください。

## 設定項目

`設定 > 拡張機能 > Claude Cost`（または `settings.json`）で変更できます。

| 設定キー | 既定 | 説明 |
| --- | --- | --- |
| `claudeCost.claudeHome` | `""`（=`~/.claude`） | データディレクトリの上書き |
| `claudeCost.useCliUsage` | `true` | 使用率%取得に公式 CLI（`claude -p "/usage"`）を使う |
| `claudeCost.claudeCliPath` | `""`（自動解決） | `claude` 実行ファイルのパス上書き |
| `claudeCost.usageRefreshIntervalSeconds` | `300` | 使用率%の取得間隔（秒・キャッシュTTL・条件付きポーリング間隔を兼ねる） |
| `claudeCost.warnThresholdPercent` | `80` | 使用率が「接近」と判定される閾値（%） |
| `claudeCost.priceOverrides` | `{}` | モデル別 API 価格の上書き |
| `claudeCost.unknownModelHandling` | `"zero"` | 価格未定義モデルの扱い（`zero`/`exclude`/`warn`） |
| `claudeCost.refreshDebounceMs` | `2000` | ファイル変更検知後、再集計するまでのデバウンス時間（ms） |
| `claudeCost.notifyOnOverage` | `true` | 使用率が接近・上限到達へ悪化した際のポップアップ通知 |
| `claudeCost.showRawModelNames` | `false` | ローカル/特殊モデル名を生表示するか |

> 使用率%は公式 `claude -p "/usage"` の実測値です。手動で上限値を設定する必要はありません。`claude` が見つからない場合は API 換算コストのみ表示されます。
